MANUAL GIT

    inicialização de repositorio

  • git init: Inicializa um novo repositório Git no diretório atual.
  • git clone [URL]: Clona um repositório Git existente a partir de uma URL.

    • gerenciamentos de alteração

  • git add [arquivos]: Adiciona arquivos à área de preparação para serem incluídos no próximo commit
  • git commit -m "[mensagem]": Cria um novo commit com os arquivos na área de preparação e uma mensagem que descreve as alterações.
  • git status: Mostra o estado dos arquivos no repositório.
  • git log: Mostra o histórico de commit

    • interação com repositorio remoto

  • git pull: Obtém as alterações mais recentes do repositório remoto e as mescla com sua ramificação atual.
  • git push: Envia suas alterações locais para um repositório remoto.
  • git remote -v: Lista os repositórios remotos configurados.
  • Git remote rm origin: Apagar a ligação com o repositório remoto
  • Git gnore: se vc quiser que o git não rastreie seu arquivo

O que é o git?

O Git é um sistema de controle de versões distribuído amplamente utilizado na indústria de desenvolvimento de software. Ele foi criado por Linus Torvalds em 2005, o mesmo criador do sistema operacional Linux, e é um software de código aberto que permite rastrear as alterações feitas em arquivos e diretórios ao longo do tempo. O Git é projetado para facilitar o trabalho colaborativo em projetos de software, mas também pode ser usado para rastrear alterações em qualquer tipo de arquivo. O Git oferece uma maneira eficaz de gerenciar versões de código, facilita a colaboração em equipe, rastreia o histórico de alterações e fornece umsistema de controle de versão sólido. É amplamente usado na indústria de desenvolvimento de software e é uma ferramenta essencial para programadores e equipes de desenvolvimento que desejam trabalhar de forma eficiente e colaborativa em projetos de software.

tabela resumindo o git

termos descrição
git Git é um sistema de controle de versões distribuído (VCS) que rastreia alterações em arquivos e diretórios ao longo do tempo. É amplamente usado no desenvolvimento de software.
repositório Um repositório Git é um local onde todas as informações sobre um projeto são armazenadas, incluindo o histórico de alterações, arquivos e diretrizes para o controle de versão.
commit Um commit é uma unidade fundamental de alteração no Git. Representa uma coleção de alterações em arquivos que foram registradas no histórico do repositório.
